Destination Journey

Visual concept. The app doesn't open — you arrive. A sunrise glow gathers over an abstract land: two soft silhouette layers whose profile is the property's identity — coastline swells, mountain ridges, dunes, a skyline, a treeline. A hairline path of light draws toward the horizon, a single point of light travels it, and where the journey ends the light blooms and becomes the brand. Everything is silhouette and light — never a literal landscape.

Design rationale. Every other variant stages the brand; this one earns it — the logo is the destination of a two-second story: place → path → arrival. It's the most emotionally loaded variant and the most personal to each property, because the land itself is a CMS token. Cinematic comes from sequence and parallax, not effects: three moving layers, one traveler, zero particles.

02

Motion specifications

journey path · quadratic bézier (14,196) → ctrl (70,164) → (50,108) in a 100×216 canvas — origin low-left, destination exactly at the emblem center. Draw via dash-offset; length computed numerically.
traveler · 8 dp point of warm white, glow 0 0 12 primary@80% · position = bézier(t) with easeInOutCubic — slow departure, glide, gentle arrival.
terrain parallax · back layer drifts ±1.2u at 0.9 rad/s; front drifts opposite ±1.5u at 0.8 rad/s — both off the master clock, never a second timer.
arrival · pulse ring 0.3→1.6 (fast-in, fade-out) + emblem scale 0.6→1.00 growing out of the traveler's final position, blur 5→0.
easing set · easeOutCubic (fades, land, emblem) · easeInOutCubic (traveler, rule, hand-off) · no springs — arrival should feel inevitable, not bouncy.
03

White-label behavior

The land is a token — environmentTheme selects a pair of silhouette profiles (back + front). Five ship at launch: coast · mountain · desert · skyline · forest. A beach property and an alpine lodge run the same code with different strings.
Themes are data, not code — each theme is two SVG path strings + nothing else. New themes (lakefront, vineyard, marina…) are added to the CMS catalog without an app release.
Silhouettes inherit the brand — terrain fills are primary mixed into the ground (18% / 10% dark; 30% / 24% light), so the land always sits in the client's palette. No separate terrain colors to configure or get wrong.
Choreography is invariant — path, traveler, arrival and text beats are identical across themes; only the profile of the land changes. QA once, ship to hundreds of properties.

06

Accessibility

Reduced motion — path, traveler and parallax are skipped; 300ms cross-fade of the settled frame (land at rest, emblem present). Shared code path with all variants.
Vestibular safety — parallax amplitude is ±1.5% of width at ≤0.15 Hz; the traveler is a single small point. No full-field motion, no zoom. Well inside WCAG 2.3.3 comfort.
Contrast — terrain sits at the bottom third; the wordmark block never overlaps a silhouette edge. Name vs ground ≥ 4.5:1 validated at ingest.
Screen readers — root announces "‹Casino name›. Loading."; the journey is purely decorative and unannounced.
Time limit — hand-off at timeline end or when Home is ready; hard cap 4s.
07

React Native · performance · developer handoff

Terrain — two react-native-svg Paths (or Skia). Theme = a pair of `d` strings from the CMS catalog; fills are computed color mixes of primary into the ground. Animate wrapper transform only, never the path data.
Journey path — one svg Path with animated strokeDashoffset; precompute length once (32-sample polyline). Traveler = a View positioned by the same bézier evaluated in the worklet — no path-following library needed.
Performance budget — 5 animated layers (2 terrain, path, traveler, emblem) + 2 one-shot (bloom, pulse). Transform/opacity/dashoffset only; single small blur via pre-blurred logo cross-fade; no particles, no filters, no full-screen effects. 60fps on Android Go-class hardware.
Battery & startup — the whole variant is ~0 extra assets (paths are strings); preload Home during the 2.6s; splash runs once per cold start, no loops.
Hand-off — emblem → header slot (scale 1→0.42) as in every variant; the land dissolves with the scene. Ground gradient persists into Home.
Authoritative math — variant === 'journey' branch in SplashScreen.dc.html (bézier, dash math, parallax, all windows). Complete 8-variant package: design_handoff_splash_system/README.md.
