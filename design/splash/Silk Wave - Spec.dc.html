Variant spec · animationVariant: "silk"

Silk Wave

Visual concept. Three ribbons of silk-light drift across a still, dark field — fabric moving in slow motion. The brightest ribbon crosses the emblem's position and the logo emerges in its wake, as if silk were sliding off it. The flow then calms: one ribbon settles into the thin accent line beneath the name. Nothing snaps, nothing bounces — the whole scene breathes at one slow tempo.

Design rationale. Silk is the material language of VIP hospitality — couture, table linens, stage curtains. Where The Threshold says arrival, Silk Wave says being attended to. It's the calmest variant in the set: no springs, no pings, only drift and reveal — suited to luxury casinos, VIP clubs and five-star resort brands. Same layout system as every variant; only the choreography changes.

02

Layout

Silk Wave uses the shared layout system — same safe areas (44 / 24 / 34 dp), same 96 dp emblem box at 50% H, same wordmark block at 60% H. Only the ribbon layer is new:

ribbon A · top 34% H · height 13% W · rotate −16° · peak opacity .50
ribbon B · top 45% H · height 15% W · rotate −10° · peak opacity .85 — crosses the emblem
ribbon C · top 57% H · height 11% W · rotate −13° · peak opacity .40
Each ribbon: 190% W wide, soft-edged (border-radius 50%, blur ≈ 10), fill = linear-gradient 90° — transparent → primary@38% → warm white 60% core → primary → transparent. Always the brand hue + white; never a second color.
Ribbons live behind the emblem and wordmark (z 4 vs 6) so type stays crisp; the top scrim keeps the status bar legible.

animationDuration scales the master clock linearly — every window keeps its proportion. Clamped to 1800–3000 ms so clients can't make it sluggish or frantic.
Ribbon color is always derived from primaryColor — clients cannot pick a separate ribbon hue, protecting the single-light-source rule.

06

Accessibility

Reduced motion — AccessibilityInfo.isReduceMotionEnabled → skip ribbons entirely; 300ms cross-fade of the settled frame, then navigate. Shared code path with all variants.
Contrast — name text vs ground ≥ 4.5:1, validated at CMS ingest. Ribbons pass behind type; peak ribbon opacity under the wordmark region (ribbon C) is capped at .40 so text never sits on a bright band.
Vestibular safety — max translation speed ≈ 38%/s of screen width, no rotation animation, no scale pulsing. Well inside WCAG 2.3.3 comfort ranges.
Screen readers — splash root is a single accessibilityLabel: "‹Casino name›. Loading." VoiceOver focus is not trapped; navigation announces Home on hand-off.
Time limit — splash never blocks: hand-off fires at timeline end or the moment Home is ready, whichever is later, capped at 4s hard.
07

React Native & handoff notes

Ribbons — three absolutely-positioned LinearGradient bands (190% W, rotated), each with its own useAnimatedStyle reading the shared clock. Softness: bake the blur into a pre-blurred PNG/Lottie ribbon asset rather than a runtime blur — runtime blur on 190%-wide layers will not hold 60fps on low-end Android.
Optional Lottie upgrade — replace the three bands with one full-screen Lottie of genuinely curved silk (same 2.2s timeline, driven by progress so scrub/duration still work). The Reanimated fallback above must ship regardless.
Sine drift — implement with Math.sin inside the worklet off the same clock; do not run a second timer.
Emblem emergence — opacity/translate/scale via clock windows; the 6→0 blur uses a pre-blurred logo copy cross-faded against the sharp one (two Images, opacity-only).
Hand-off — same shared-element contract as all variants: emblem → header slot (scale 1→0.42), ground persists, scene fades in the final 200ms.
Authoritative math — the variant === 'silk' branch in SplashScreen.dc.html; port `seg / eo / eio / lerp` + the sine term 1:1 into the worklet. Full package: design_handoff_splash_system/README.md.
