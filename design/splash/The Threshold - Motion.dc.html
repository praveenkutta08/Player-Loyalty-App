03

Motion principles

One light source
Every element is lit and led by the horizon. Motion radiates from that single line — nothing enters from nowhere.
Offset & overlap
Beats are staggered and gently overlapping — never a synchronized "everything at once." Reads as choreography, not a loader.
One hero moment
The emblem's spring is the only accented beat. Everything else is quiet support — restraint is the premium signal.
Transform-only
Opacity, translate, scale, blur — all GPU-composited on the UI thread. No width/height/layout, so it holds 60 fps on low-end devices.
The end is the beginning
The final frame is Home's first frame. Continuity replaces the cut.

05

One implementation, every brand

The timeline above is fixed code — a set of shared Reanimated worklets. Nothing about the choreography is per-brand. The CMS supplies only values, which flow in as the animation's inputs and driven styles.

const s = useSplashConfig(); // CMS
const t = useSharedValue(0);

useEffect(() => {
  t.value = withTiming(1, {
    duration: 2400,
    easing: Easing.linear // beats eased inside
  }, () => runOnJS(goHome)());
}, []);

// one worklet, per-element windows
const emblem = useAnimatedStyle(() => ({
  opacity: seg(t.value, .65, 1.15),
  transform: [{ scale: spring01(t) },
          { translateY: rise(t) }]
}));

<Splash variant={s.animationVariant}
      logo={s.logo} name={s.casinoName}
      primary={s.primaryColor}
      background={s.backgroundStyle} />

One shared clock t; each element reads its own window off it. Brands change the inputs, never the worklets. prefers-reduced-motion collapses to a 300ms cross-fade of the same final frame.
