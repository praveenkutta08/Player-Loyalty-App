All eight variants share the layout system from the spec — same safe areas, same emblem box, same wordmark block, same hand-off. They differ only in choreography, so a client can switch splash at any time from the CMS with zero engineering. The JSON below is live — it reflects the variant selected above.

Client picker, not a fork
The CMS exposes the variants as a visual picker (thumbnails like the row above). Choosing one writes a single string — no build, no branch, no per-client bundle.
Safe fallback
Unknown or missing variant → horizon. A failed Lottie fetch (seal) falls back to the static logo with the horizon timing.
Reduced motion
If the OS requests reduced motion, every variant collapses to the same 300ms cross-fade of its final frame — one shared code path.
Tiering opportunity
Variants can be sold as a premium CMS feature: base plans get horizon, premium plans unlock the full set.
